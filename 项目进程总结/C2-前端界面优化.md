# C2 前端界面优化

## 本阶段目标

在前端页面基础搭建完成后，对界面进行系统性优化，提升可读性、视觉层次感和交互体验。

## 优化内容

### 1. 字体与间距

| 区域 | 修改前 | 修改后 |
|---|---|---|
| 全局基准字号 | 14px | 15px |
| Ant Design 标题 | 默认 | H1-32 / H2-26 / H3-22 / H4-18 / H5-16 |
| 侧边栏导航 | 40px 高，默认字号 | 44px 高，15px |
| 合同原文 | 13px，默认行高 | 16px，行高 1.8 |
| 工作区 padding | 24px | 28px |
| 登录面板 | 420px / 28px padding | 440px / 36px padding |

### 2. 视觉层次感

- Dashboard 统计卡片：3px 彩色顶边（灰蓝/琥珀/暗红）+ 悬浮阴影
- 风险卡片：4px 左边框色条对应风险等级（高/中/低）
- 审查详情三栏：左栏和右栏加浅灰底色 `#fafbfc` 与中栏形成视觉分隔
- 审批中心 PENDING 标签：2s 呼吸式脉冲动画
- 全局颜色降饱：统计数值/标签使用 `#b85c5c` / `#b8864e` / `#5c7c99` 等沉稳色调

### 3. 加载与空状态

| 页面 | 修改前 | 修改后 |
|---|---|---|
| Dashboard | 整页 Spin 转圈 | 骨架屏卡片 + 加载完渐变显示 |
| Dashboard 空状态 | 纯文字 | 引导按钮（去审查合同 / 上传第一份合同） |
| Dashboard 失败 | 静默空数据 | 错误提示 + 重新加载按钮 |
| 审查详情 | 整页 Spin | 三栏骨架屏 |
| 知识库问答 | 单调搜索框 | 示例问题 Tag 点击即搜 + 检索中骨架 |

### 4. 格式与中文显示

- 合同管理页：状态列改为中文（已上传/已解析/解析失败/已归档）
- 用户权限页：角色名中文化 + 颜色区分（管理员/法务主管/法务人员/业务人员/只读用户）
- 工具配置页：风险等级中文化（高风险/中风险/低风险）
- 规则库管理：操作列宽度调整，按钮溢出修复

### 5. 知识库问答 — 对话式 UI

- 用户问题：右侧蓝色圆角气泡 + 用户头像
- AI 检索结果：机器人头像 + "检索到 N 条" 提示 + 每 chunk 一张卡片
- 卡片内容直接显示，底部标注文档来源和相似度
- 示例问题 Tag 可点击直接提问
- 输入框固定底部，Enter 发送

### 6. Dashboard 增强

- 最近审查任务列表：显示真实合同名称（替换"审查 #id"）
- 每个任务加进度条，根据审查状态自动计算进度百分比
