# A2-数据库功能表/日志表建立

## 本阶段目标

在 V1（基础表）和 V2（RBAC 种子）之上，补齐合同审查主链路所需的业务功能表和日志审计表。

## 已完成任务

```text

DB#04 合同审查任务表
DB#05 合同解析与条款表
DB#06 风险结果表

DB#08 Agent 步骤与状态流转表
DB#09 LLM 调用日志表
DB#10 工具调用日志表
DB#11 RAG 检索日志表
```

## 实施说明

### 业务功能表（V3-V6）

| 迁移 | 新建表 | 说明 |
| --- | --- | --- |
| V3 | — | 为 contract / contract_review 补充 9 个查询索引 |
| V4 | contract_clause | 存储 LLM 抽取的 12 类结构化条款（JSONB），关联 contract |
| V5 | clause_risk | 存储 10 种风险类型的结果、原因、建议、引用规则、审批标记 |
| V6 | approval_request / approval_history | Human-in-the-loop 审批，支持审查级/条款级/报告级审批 |

### 日志审计表（V7）

V7 为 V1 中已有的四张日志表统一补齐索引（共 16 个），覆盖按审查任务追溯、按类型/状态筛选、按时间排序等核心查询路径：

```text
agent_step、agent_state_transition_log
llm_call_log
tool_call_log
retrieval_log
```
