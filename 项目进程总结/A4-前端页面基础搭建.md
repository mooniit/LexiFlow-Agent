# A4 前端页面基础搭建

## 本阶段目标

完成 LexiFlow Agent 前端首批页面（FE#01-FE#05），从单文件原型重构为 react-router 多页面架构，对接后端 REST API 和 SSE 实时事件流。

## 已完成任务

- FE#01 登录页：JWT 登录表单、token 持久化、已登录自动跳转 Dashboard。
- FE#02 工作台 Dashboard：审查任务统计卡片、最近审查列表、Agent 执行时间线、合同概览。
- FE#03 合同列表页：合同分页表格、名称/客户搜索、状态筛选、行内查看/审查/删除操作。
- FE#04 合同上传页：拖拽上传 txt/docx、合同元数据表单（名称/类型/金额/客户）。
- FE#05 合同审查详情页：三栏布局（合同原文 | 风险点与修改建议 | Agent 时间线 + SSE 实时事件），支持 SSE 实时状态推送。

## 架构变更

- 引入 react-router-dom v6，采用嵌套布局路由：`LoginPage` 独立于 Layout 外，`DashboardPage`、`ContractListPage`、`ContractUploadPage`、`ContractReviewPage` 共享 `AppLayout` 壳（Sider + Header + Content）。
- API 层拆分为 `client.ts`（fetch 封装 + JWT 注入）、`auth.ts`、`contract.ts`、`review.ts` 四个模块。
- 路由映射：

```
/login              -> LoginPage
/dashboard          -> DashboardPage
/contracts          -> ContractListPage
/contracts/upload   -> ContractUploadPage
/contracts/:id      -> ContractReviewPage (?reviewId=)
```

## Bug 修复
