# DB#06-风险结果表

## 任务目标

创建 clause_risk 表，保存条款风险识别结果，包括风险等级、类型、原因、修改建议、引用规则和审批标记。

## 对应总览编号

```text
DB#06
```

## 涉及模块

```text
database: clause_risk
backend: review, agent
```

## 本次实现

`V5__create_clause_risk.sql` 新建表，包含：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGSERIAL PK | 主键 |
| review_id | BIGINT FK | 关联审查任务 |
| clause_id | BIGINT FK | 关联条款（可空） |
| risk_level | VARCHAR(20) | LOW / MEDIUM / HIGH |
| risk_type | VARCHAR(64) | 风险类型枚举 |
| clause_name | VARCHAR(200) | 条款名称 |
| clause_text | TEXT | 条款原文 |
| reason | TEXT | 风险原因分析 |
| suggestion | TEXT | 修改建议 |
| evidence_rules | JSONB | 引用规则依据数组 |
| requires_approval | BOOLEAN | 是否需要人工审批 |
| review_status | VARCHAR(32) | 风险处理状态 |

支持的风险类型：PAYMENT_TERM_TOO_LONG / UNLIMITED_LIABILITY / MISSING_DATA_PROTECTION / NON_STANDARD_JURISDICTION / AUTO_RENEWAL_RISK / ONE_SIDED_TERMINATION / CONFIDENTIALITY_WEAK / IP_OWNERSHIP_UNCLEAR / HIGH_CONTRACT_AMOUNT / MISSING_TERMINATION_CLAUSE

## 验收标准

- 表结构与 V1 审计字段规范一致
- review_id 关联 contract_review，clause_id 可空关联 contract_clause
- evidence_rules JSONB 支持规则引用溯源
- requires_approval 驱动审批流程触发
- 索引覆盖常用查询字段
