# 企业合同与合规审查 Agent 项目功能说明

## 1. 项目名称

```text
LexiFlow Agent - 企业合同与合规审查 Agent
```

## 2. 项目定位

本项目是一个面向企业销售、采购、法务团队的合同审查 Agent Web 应用。

核心目标：

```text
上传一份合同，Agent 自动审查风险，必要时进入法务审批，最后生成可追溯的审查报告。
```

项目不定位为通用聊天机器人，而是一个审查型、流程型、可追溯的企业 Agent 系统。

---

## 3. 目标用户与角色

### 1. 业务人员 BUSINESS_USER

典型用户：销售、采购、商务人员。

权限范围：

- 上传合同
- 发起合同审查
- 查看自己提交的审查任务
- 查看审查结果和修改建议
- 根据法务意见重新提交合同

### 2. 法务人员 LEGAL_REVIEWER

权限范围：

- 查看分配给自己的审查任务
- 查看 Agent 风险识别结果
- 补充法务意见
- 处理一般风险
- 标记风险处理状态

### 3. 法务主管 LEGAL_MANAGER

权限范围：

- 审批高风险合同或高风险条款
- 批准、驳回、要求修改或转人工处理
- 查看审批历史
- 查看高风险统计

### 4. 管理员 ADMIN

权限范围：

- 用户管理
- 角色权限管理
- 合规规则库管理
- 合同模板管理
- 审查工具配置
- 系统配置管理

### 5. 只读用户 VIEWER

权限范围：

- 查看授权范围内的审查记录
- 查看审查报告
- 查看知识库问答结果

## 4. 核心业务链路

### 链路 1：合同自动审查

这是项目主链路。

流程：

```text
用户上传合同
 -> 创建审查任务
 -> 解析合同文本
 -> 抽取关键条款
 -> 检索公司合规规则库
 -> Agent 识别风险点
 -> 生成风险等级和修改建议
 -> 判断是否需要人工审批
 -> 生成审查报告
 -> 保存 Agent Trace
```

前端通过 SSE 实时展示过程：

```text
合同上传完成
合同解析中
条款抽取中
规则检索中
风险分析中
等待审批
报告生成中
审查完成
```

### 链路 2：高风险条款人工审批

合同审查中，Agent 不应直接对高风险合同给出最终通过结论。

触发审批的典型条件：

- 总体风险等级为 HIGH
- 合同金额超过阈值
- 存在无限责任条款
- 付款周期超过公司标准
- 缺失数据保护条款
- 存在非标准争议解决条款
- 存在单方解除权
- 存在自动续约风险

流程：

```text
Agent 识别高风险
 -> 创建 approval_request
 -> 审查任务进入 WAITING_APPROVAL
 -> 法务主管查看风险说明和依据
 -> 批准 / 驳回 / 要求修改 / 转人工处理
 -> RabbitMQ 发布审批事件
 -> Agent 根据审批结果继续生成最终报告
```

### 链路 3：合规知识库问答

用户可以直接向合规知识库提问。

示例问题：

```text
公司对付款周期有什么要求？
什么情况下需要法务主管审批？
销售合同中的数据保护条款必须包含哪些内容？
客户要求无限赔偿责任时应该怎么处理？
自动续约条款有什么风险？
```

回答要求：

- 基于合规知识库检索结果回答
- 展示引用来源
- 不编造规则
- 必要时建议用户发起合同审查

## 5. 功能模块

## 5.1 登录与权限模块

功能：

- 用户登录
- JWT 鉴权
- 角色权限控制
- 接口权限控制
- 工具权限控制
- 知识库访问权限控制

角色：

```text
ADMIN
LEGAL_MANAGER
LEGAL_REVIEWER
BUSINESS_USER
VIEWER
```

项目价值：

- 体现 Spring Security + JWT + RBAC
- 体现企业 Agent 的权限边界
- 防止 Agent 越权访问知识库或调用高风险工具

## 5.4 合同解析模块

功能：

- 读取合同文件
- 提取合同纯文本
- 保存解析结果
- 记录解析步骤
- 解析失败时记录错误原因

第一版重点：

- 文本解析正确
- 能支撑后续条款抽取
- 不追求复杂版式还原

项目价值：

- 作为合同审查链路的入口
- 为后续条款抽取和风险识别提供基础数据

项目价值：

- 体现 LLM 结构化输出能力

功能：

- 输入合规问题

项目价值：

- 体现 RAG 问答能力

- 让规则库价值更直观
