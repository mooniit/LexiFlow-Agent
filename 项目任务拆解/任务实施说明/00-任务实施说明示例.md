# G#01-登录与权限模块

> 这是实施说明示例文件。后续每个任务都可以按 `任务编号-任务名称.md` 的格式新建文件，例如 `G#07-自研Agent状态机.md`、`DB#04-合同审查任务表.md`、`DEP#03-DockerCompose本地环境.md`。

## 任务目标

实现用户登录、JWT 鉴权、RBAC 角色权限控制，并为 Agent 工具调用和知识库访问控制提供用户上下文。

## 对应总览编号

```text
G#01
```

## 涉及模块

```text
backend: security, user, common, config
database: user / role / permission / user_role / role_permission
frontend: 登录页、权限路由
```

## 实施步骤

1. 建立用户、角色、权限相关表结构和 Flyway 迁移。
2. 实现用户实体、Mapper、Service 和登录接口。
3. 接入 Spring Security，配置 JWT 过滤器和无状态认证。
4. 实现角色权限加载与接口授权。
5. 在登录响应中返回 token、用户信息和角色信息。
6. 为 Agent 执行上下文提供当前用户、角色、部门和权限集合。
7. 编写单元测试和接口测试。

## 验收标准

- 支持 `ADMIN`、`LEGAL_MANAGER`、`LEGAL_REVIEWER`、`BUSINESS_USER`、`VIEWER` 五类角色。
- 未登录用户不能访问受保护接口。
- 不同角色只能访问授权范围内接口。
- 后端能从 JWT 中解析当前用户身份和权限。
- 前端登录后可保存 token 并访问受保护页面。
