# A3 主要功能构建

## 本阶段目标

完成合同审查 MVP 主链路的主要后端能力建设，覆盖合同管理、文本解析、审查任务、REST API、SSE 事件流、条款抽取、规则库、RAG 检索和风险识别。

## 已完成任务

- G#04 合同管理模块：支持 txt/docx 上传、合同列表、详情、原文、解析、归档和逻辑删除。
- G#05 合同文本解析模块：支持 txt 文本解析入库，docx 暂返回明确失败原因。
- G#06 合同审查任务模块：支持创建审查、状态流转、步骤记录、取消和失败重跑。
- G#09 REST API 接口：补齐合同、审查任务、风险、知识库和检索相关接口。
- G#10 SSE 实时事件流：按 reviewId 推送审查状态和步骤事件。
- G#13 条款抽取模块：基于关键词和正则抽取主体、金额、付款周期、期限、责任、保密、知识产权、数据保护、争议解决、解除、赔偿上限、自动续约等条款。
- G#14 合规规则库模块：支持知识库创建、规则文档上传、txt 切分入库和 chunk 查询。
- G#15 RAG 检索模块：基于规则库 chunk 的关键词相似度检索，并记录 retrieval_log。
- G#16 风险识别模块：识别付款周期过长、无限责任、缺失数据保护、自动续约等风险，生成原因、建议、引用依据和审批标记。

## 关键实现

- 新增 JSONB TypeHandler 与 JSON 字符串工具，支撑半结构化字段写入 PostgreSQL。
- 新增合同、条款、审查、风险、知识库、文档 chunk、检索日志等模型与 Mapper。
- 将审查主链路从占位流程升级为：解析合同 -> 抽取条款 -> 检索规则 -> 识别风险 -> 写入结果摘要 -> SSE 推送。
- 对齐 master 上已有 DB#03-DB#07 迁移，避免重复创建 contract_clause 与 clause_risk 表。

## 验证情况

- 在开始本阶段前，`mvn test` 已确认通过，7 个测试全绿。
- 新功能实现后，`mvn "-Dmaven.test.skip=true" package` 通过，主应用可编译打包。
- 新功能实现后，`mvn test` 在本地再次复现 testCompile classpath 异常：测试编译器未识别已编译的主类。该问题此前可通过环境/构建状态恢复，并非当前业务代码的主编译错误。

## 后续建议

- 引入 Apache POI 或独立解析服务，补齐 docx 真实文本解析。
- 将 RAG MVP 的关键词检索升级为 pgvector 检索；当前 mock embedding 为 16 维，数据库向量字段为 1536 维，暂未写入 embedding。
- 将条款抽取与风险识别从启发式规则升级为 LLM structured output + 规则引擎混合模式。
- 继续推进 G#17 高风险审批触发、G#18 人工审批、G#20 审查报告和 G#21 Agent Trace。

## 2026-05-29 后续任务推进

- G#17 高风险审批触发：已接入审查主链路。风险分析后若出现 HIGH 风险或 `requiresApproval=true`，审查任务进入 `WAITING_APPROVAL`，并创建待处理审批请求。
- G#18 人工审批模块：新增审批请求、审批历史实体与 Mapper，新增 `/approvals` 列表、详情、历史、通过、驳回、要求修改、升级处理接口。审批通过后恢复报告生成；驳回或要求修改后任务失败并记录原因。
- G#20 审查报告模块：新增 `/reviews/{id}/report`，返回审查任务、风险明细与结果摘要；审批通过后报告摘要会标记 `approvalPassed=true`。
- G#21 Agent Trace 模块：新增 `/reviews/{id}/trace`，返回审查任务、步骤记录与状态流转记录。
- 风险规则增强：新增高额合同 `HIGH_CONTRACT_AMOUNT` 风险识别，金额达到 100 万阈值时自动要求审批；高风险或需审批风险统一触发人工审批。
