# DB#12-知识库与文档表

## 任务目标

完善 knowledge_base 与 knowledge_document 表，支撑合规规则库、合同模板和审查规范的存储与权限控制。

## 对应总览编号

```text
DB#12
```

## 涉及模块

```text
database: knowledge_base, knowledge_document
backend: rag, security
```

## 本次实现

- `V1__init_schema.sql` 已创建两张基础表。
- `V8__enhance_knowledge_and_jsonb.sql` 补充索引：

**knowledge_base**：支持 PUBLIC/PRIVATE/DEPARTMENT 三级可见性，allowed_roles JSONB 控制角色级访问，department_id 控制部门级访问。

| 新增索引 | 类型 |
| --- | --- |
| idx_knowledge_base_visibility | BTREE |
| idx_knowledge_base_status | BTREE |
| idx_knowledge_base_department_id | BTREE |
| idx_knowledge_base_allowed_roles | GIN |

**knowledge_document**：关联知识库，按 document_type 区分 POLICIES/TEMPLATE/GUIDELINE/CASE。

| 新增索引 | 类型 |
| --- | --- |
| idx_knowledge_document_kb_id | BTREE |
| idx_knowledge_document_type | BTREE |
| idx_knowledge_document_status | BTREE |

## 验收标准

- 支持按可见性、部门、角色三级权限过滤
- GIN 索引支持 JSONB allowed_roles 包含查询
- 与 G#03 知识库访问权限控制设计对齐
